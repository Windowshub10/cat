DHCP
Step 1: Check if the DHCP Package is Installed

Cmd: rpm -qa | grep dhcp

If not installed then: go the media -> RHEL... -> Packages file

Cmd: rpm -ivh dhcp*

Step 2: Configure a Static IP for the server.

Cmd: setup

Select network configuration -> run tool

Select device configuration.

Edit your lan card (i.e eth0)

Set the Static IP

uncheck the use DHCP option

In the Static IP filed, ent 192.168.1.3

In the Netmask filed enter 255.255.255.0

Save and Exit all.

Step 3:

Cmd: ifdown eth0, ifup eth0

Step 4: Prepare the DHCP Configuration File

Cmd: cp /usr/share/doc/dhcp*/dhcpd.conf.sample /etc/dhcp/dhcpd.conf

press 'y'

Step 5: Edit the dhcpd.conf file

Cmd: gedit /etc/dhcp/dhcpd.conf

Option domain-name "tyit.com";

Option domain-name-servers server.tyit.com;

Uncomment the authoritative;

Add the following block:

subnet 192.168.1.0 netmask 255.255.255.0 { 
	range 192.168.1.10 192.168.1.20; 
}

save the file

Step 6: Start and Enable the DHCP service.

Cmd: service dhcpd start

chkconfig dhcpd on
service iptables stop
Setenforce 0

Now client:
Step 1: Close dhcp the server

Cmd: service dhcpd stop

Step 2: Clone the machine of server

(Create full clone)

Step 3: First Power on server VM and then the client VM.

Step 4: Change the network static to DHCP (Automatic)

Cmd: gedit /etc/sysconfig/network-scripts/ifcfg-eth0

change the BootProto to dhcp

Step 5: restart the Network service.

Cmd: service network restart

Step 6: ifconfig.

Note: If not get IP address as in range then go to the network adapter -> Advanced

In Advanced Copy the Real Mac address and then go to the VM:

Cmd: gedit /etc/sysconfig/network-scripts/ifcfg-eth0

in this paste the new Mac address in HWADDR

Save it

then do service restart

Cmd: ifconfig
